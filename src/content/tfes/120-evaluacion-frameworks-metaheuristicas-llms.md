## Descripción

Existen numerosas bibliotecas para optimización y metaheurísticas desarrolladas en distintos lenguajes de programación. Aunque habitualmente se comparan desde el punto de vista del rendimiento o del número de algoritmos implementados, resulta mucho más difícil evaluar aspectos como su facilidad de uso, extensibilidad, calidad de la documentación o esfuerzo necesario para incorporar nuevas funcionalidades.

Este trabajo propone utilizar modelos de lenguaje como herramienta para evaluar estos frameworks desde la perspectiva de un desarrollador. Para ello se diseñará una metodología experimental reproducible basada en una colección de tareas representativas, como resolver nuevos problemas de optimización, implementar operadores, incorporar algoritmos o extender la biblioteca. El estudio permitirá comparar distintos frameworks utilizando métricas objetivas relacionadas con el esfuerzo de desarrollo y la calidad del código generado.

Además del análisis experimental, el trabajo podrá explorar cómo influyen diferentes modelos de lenguaje, estrategias de prompting o disponibilidad de documentación en los resultados obtenidos, proporcionando una nueva forma de evaluar herramientas software.

## Posibles líneas de trabajo

- Seleccionar frameworks representativos.
- Diseñar una metodología de evaluación reproducible.
- Definir tareas representativas para los modelos de lenguaje.
- Automatizar la ejecución de experimentos.
- Comparar diferentes modelos y estrategias de prompting.
- Analizar la calidad del código generado y el esfuerzo requerido.
