## Descripción

Los proyectos software evolucionan continuamente incorporando nuevas funcionalidades, corrigiendo errores o modificando su arquitectura. Como consecuencia, las pruebas existentes pueden dejar de ser válidas o necesitar modificaciones para seguir comprobando correctamente el comportamiento esperado.

Este trabajo propone desarrollar una herramienta capaz de analizar automáticamente los cambios realizados sobre el código fuente, identificar qué pruebas deberían modificarse y generar propuestas de actualización utilizando modelos de lenguaje junto con técnicas de análisis estático y validación automática.

El objetivo será construir un proceso completo que permita detectar las pruebas afectadas, proponer modificaciones, verificar automáticamente las soluciones obtenidas y comparar distintas estrategias de generación sobre un conjunto de proyectos reales.

## Líneas principales

- Análisis de cambios entre versiones de un proyecto software.
- Identificación automática de pruebas potencialmente afectadas.
- Desarrollo de estrategias para adaptar automáticamente las pruebas.
- Integración de análisis estático, ejecución de pruebas y modelos de lenguaje.
- Comparación de diferentes estrategias y modelos de generación.
- Evaluación experimental utilizando proyectos y benchmarks públicos.
