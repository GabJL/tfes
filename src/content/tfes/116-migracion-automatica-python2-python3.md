## Descripción

Aunque Python 2 dejó de mantenerse hace años, todavía existen numerosos proyectos que continúan utilizándolo debido al elevado coste de migración. Las herramientas automáticas existentes permiten realizar parte de la conversión, pero con frecuencia requieren una importante intervención manual para resolver incompatibilidades, dependencias o cambios de comportamiento.

Este trabajo propone desarrollar una plataforma que automatice el proceso completo de migración combinando herramientas tradicionales, análisis estático e inteligencia artificial. El objetivo no es únicamente generar código Python 3, sino construir un proceso robusto que valide automáticamente cada transformación utilizando compilación, ejecución de pruebas y otras estrategias de verificación.

El estudiante podrá explorar diferentes estrategias de migración, desde procesos secuenciales hasta arquitecturas basadas en agentes especializados, evaluando distintos modelos de lenguaje y protocolos de interacción sobre un conjunto de proyectos reales.

## Líneas principales

- Diseño de una arquitectura para automatizar el proceso de migración.
- Integración de herramientas tradicionales de conversión con modelos de lenguaje.
- Desarrollo de mecanismos automáticos de validación mediante compilación y ejecución de pruebas.
- Implementación de distintas estrategias de interacción con el LLM (iterativa, basada en agentes, etc.).
- Comparación de diferentes modelos de lenguaje y estrategias de migración.
- Evaluación experimental sobre proyectos reales.

## Qué aprenderás

- Ingeniería del software.
- Refactorización automática.
- Análisis estático de código.
- Contenedores y automatización de procesos.
- Integración de modelos de lenguaje en herramientas software.
- Diseño experimental y evaluación de herramientas.
